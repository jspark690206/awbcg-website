# 기창원동문회수첩 모바일 앱 개발 계획서

> **문서 버전**: v1.0  
> **작성일**: 2026-05-12  
> **참조**: 기창원동문회수첩.pptx (화면 설계서)

## 1. 프로젝트 개요

### 1.1 배경 및 목적

기창원동문회 회원 상호 간 네트워킹 및 정보 공유를 위해 운영 중인 웹 시스템(`http://technoent.awbcg.co.kr`)을 모바일 앱으로 전환한다. 스마트폰 중심의 사용 환경에 맞게 접근성을 높이고, 실시간 알림·채팅·위치 기반 기능 등 모바일 특화 UX를 제공하는 것이 목적이다.

### 1.2 앱 명칭

- **한국어**: 부산대학교 기술창업대학원 수첩
- **영문**: Busan National University Graduate School of Technology Entrepreneurship Alumni Association Notebook

### 1.3 지원 플랫폼

| 플랫폼 | 버전 |
|---|---|
| iOS | 15.0 이상 |
| Android | 8.0(API 26) 이상 |

### 1.4 개발 방식

**React Native (Expo)** — 단일 코드베이스로 iOS/Android 동시 지원, 빠른 개발 속도.

## 2. 주요 기능 정의

### 2.1 화면 구성 (기획서 기반)

기존 웹 화면 설계서에서 도출된 핵심 기능은 다음과 같다.

#### 인증
- 로그인 / 로그아웃
- 아이디 변경
- 프로필 수정 (마이페이지)

#### 회원 디렉토리
- 전체 회원 목록 조회
- 회원 검색 (이름·근무지 등)
- 회원 상세 프로필 조회 (성명, 근무지명 등)
- 산업별 회원 목록 조회 (예: 정보서비스·통신업)

#### 커뮤니케이션
- 1:1 채팅
- 메모 작성·목록·조회
- 공지사항 조회 (미확인 공지 건수 뱃지)
- 이메일 연동 (Outlook)
- SMS 문자 발송

#### 경력·전문성 관리 (경력관리)
회원이 본인의 컨설팅 전문 분야를 등록·관리한다.

| 대분류 | 설명 |
|---|---|
| 창업·폐업 컨설팅 | 시장 조사, BMC 작성, 폐업 절차·절세 등 |
| 전략 컨설팅 | 시장·경쟁사 분석, 가설 검증, 워크숍 |
| 운영·프로세스 컨설팅 | 생산·공급망·구매·품질·백오피스 효율화 |
| IT·디지털(스마트팩토리) | MES, 예측 유지보수, WMS, ESG·에너지 |
| 인사·조직(HR) | 조직 구조, 성과 보상, 인재 육성, 노사관리 |
| 재무·회계·M&A | 재무 진단, 원가 분석, M&A 실사, 세무 자문 |
| 마케팅·영업 | 고객 분석, 브랜드, 디지털 마케팅, CRM |
| ESG | 전략 수립, 공시, 인증(EcoVadis·ISO), 교육 |
| 부동산 | 개발 타당성, 투자 운용, 임대 전략, 인허가 |
| 리스크관리 | ERM 체계, 금융·운영 리스크, 위기 대응 |
| 기타 | 그 외 전문 분야 |

- 대분류 선택 후 세부 경력 내역 작성
- 경력 상세 이력 등록·수정

#### 캘린더
- 월별 보기 (전월·익월 이동)
- 일정·이벤트 등록 (확장 기능)

## 3. 기술 스택

### 3.1 프론트엔드 (모바일 앱)

| 항목 | 선택 |
|---|---|
| 프레임워크 | React Native (Expo SDK 51+) |
| 언어 | TypeScript |
| 상태 관리 | Zustand |
| 네비게이션 | Expo Router (file-based) |
| UI 컴포넌트 | React Native Paper |
| 채팅 UI | React Native Gifted Chat |
| 캘린더 | react-native-calendars |
| 폼 | React Hook Form + Zod |
| HTTP 클라이언트 | Axios |

### 3.2 백엔드 / BaaS

| 항목 | 선택 | 이유 |
|---|---|---|
| 인증 | Firebase Auth | 소셜 로그인·이메일 모두 지원 |
| 데이터베이스 | Firebase Firestore | 실시간 동기화, 채팅·공지 적합 |
| 파일 저장 | Firebase Storage | 프로필 사진, 첨부파일 |
| 푸시 알림 | Expo Notifications + FCM | 미확인 공지 뱃지, 채팅 알림 |
| SMS 연동 | 카카오 알림톡 or NHN Cloud SMS | 문자 발송 기능 |

> 기존 웹(`technoent.awbcg.co.kr`) API가 있다면 별도 REST API 연동으로 대체 가능.

### 3.3 개발 환경

| 항목 | 도구 |
|---|---|
| 버전 관리 | Git (GitHub) |
| CI/CD | GitHub Actions + EAS Build |
| 배포 | Apple App Store / Google Play Store |
| 코드 품질 | ESLint + Prettier |

## 5. 데이터 모델 (Firestore)

### `users` 컬렉션

```
users/{uid}
  - displayName: string        // 성명
  - email: string
  - phone: string              // 연락처 (SMS 발송용)
  - workplace: string          // 근무지명
  - industry: string           // 업종 (예: 정보서비스·통신업)
  - profileImageUrl: string
  - consultingAreas: string[]  // 전문 컨설팅 분야 목록
  - createdAt: timestamp
  - lastLoginAt: timestamp
```

### `careers` 컬렉션

```
careers/{careerId}
  - uid: string                // 작성자 uid
  - category: string           // 대분류 (창업·폐업 컨설팅 등)
  - title: string              // 경력 제목
  - detail: string             // 상세 내역
  - period: { from, to }       // 기간
  - createdAt: timestamp
```

### `notices` 컬렉션

```
notices/{noticeId}
  - title: string
  - content: string
  - authorUid: string
  - readBy: string[]           // 열람한 uid 목록 (미확인 뱃지 계산용)
  - createdAt: timestamp
```

### `chats` 컬렉션

```
chats/{chatId}
  - participants: string[]     // [uid1, uid2]
  - lastMessage: string
  - lastMessageAt: timestamp

chats/{chatId}/messages/{msgId}
  - senderUid: string
  - text: string
  - createdAt: timestamp
```

### `memos` 컬렉션

```
memos/{memoId}
  - authorUid: string
  - targetUid: string          // 메모 대상 회원 uid
  - content: string
  - createdAt: timestamp
```

## 6. 개발 일정 (로드맵)

| 단계 | 기간 | 주요 내용 |
|---|---|---|
| **Phase 1** — 환경 구축 | 1주 | Expo 프로젝트 초기화, Firebase 연동, Git 세팅, 디자인 시스템 기초 |
| **Phase 2** — 인증·회원 | 2주 | 로그인/로그아웃, 마이페이지, 프로필 수정, 아이디 변경 |
| **Phase 3** — 회원 디렉토리 | 2주 | 전체 목록, 회원 검색, 상세 프로필, 산업별 필터 |
| **Phase 4** — 경력·전문성 | 2주 | 대분류 선택, 경력 등록·수정, 이력 상세 작성 |
| **Phase 5** — 공지사항 | 1주 | 공지 목록·상세, 미확인 뱃지, 푸시 알림 |
| **Phase 6** — 채팅·메모 | 2주 | 1:1 채팅, 메모 작성·목록, 실시간 업데이트 |
| **Phase 7** — 캘린더·이메일·SMS | 1주 | 월별 캘린더, Outlook 연동, 문자 발송 |
| **Phase 8** — QA·베타 테스트 | 2주 | 기능 검증, 성능 최적화, 사용자 피드백 수집 |
| **Phase 9** — 스토어 배포 | 1주 | App Store / Play Store 심사 제출, 출시 |
| **합계** | **약 14주** | |

## 8. 리스크 및 대응 방안

| 리스크 | 영향도 | 대응 |
|---|---|---|
| 기존 웹 DB 마이그레이션 필요 | 높음 | 기존 API 재활용 또는 일괄 데이터 이전 스크립트 작성 |
| App Store 심사 지연 | 중간 | 심사 제출 2주 전 코드 완성, 가이드라인 사전 검토 |
| 회원 데이터 개인정보 보호 | 높음 | 개인정보처리방침 수립, Firebase 보안 규칙 강화 |
| 실시간 채팅 비용 증가 | 중간 | Firestore 읽기 쿼리 최적화, 무한 스크롤 페이징 |
| 기존 회원의 앱 전환 거부 | 낮음 | 웹·앱 병행 운영, 단계적 마이그레이션 안내 |
